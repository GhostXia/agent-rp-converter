# 导入审计指南：从外部卡片到纯 Skill

> 本指南适用于 agent 将 SillyTavern、AIRP、或其他格式的外部角色卡/预设/世界书转换为纯 Skill 时的自动审计流程。

---

## 审计原则

**核心定位**：纯 Skill 方案是**纯文字、无状态、轻量级**的 RP 演绎。任何需要动态计算、持久化存储、跨会话记忆或多代理协同的机制，都必须**自动剔除**。

**agent 职责**：不提示用户逐一确认，不保留待降级，**直接删除**不可实现内容。如果剔除后卡片内容不足 50%，需在开头注明："本卡片已大幅精简，原卡包含大量状态/机制系统，超出纯 Skill 范围。"

---

## 自动剔除清单（不可实现内容）

| 类型 | 典型示例 | 处理方式 |
|------|---------|---------|
| **数值状态系统** | HP/MP/好感度/经济/经验值动态变化、属性数值表 | 整段删除 |
| **骰子/战斗系统** | 1d20、伤害计算、技能冷却表、回合规则 | 整段删除 |
| **经济/时间系统** | 金币增减、日期推进、季节变化机制 | 整段删除 |
| **文生图提示词** | Stable Diffusion tags、NSFW 描述块、图片生成指令 | 整段删除 |
| **HTML/CSS/前端** | 状态栏、进度条、颜色代码、UI 布局、地图 | 整段删除 |
| **ST 宏/运行时** | `{{char}}`、`{{user}}`、`<>`、条件脚本、替换语法 | 替换为静态文本或删除 |
| **JSON Patch / 状态更新** | 结构化数据交换、状态 diff 指令 | 整段删除 |
| **COT 标签** | `</thinking>`、`<reasoning>`、显式步骤拆解 | 删除（除非角色在思考） |
| **多 agent / NPC 协同** | 多角色并行调度、subagent 分工、阵营隔离 | 整段删除 |
| **秘密边界/视角隔离** | "仅当 X 在场时显示"、不同角色知道不同信息 | 删除门控，保留内容作为静态条目 |
| **记忆/归档指令** | "记住用户名字"、剧情日志自动归档 | 整段删除 |
| **可执行代码** | JavaScript、Python、条件逻辑、循环 | 整段删除 |
| **采样参数** | temperature、top_p、frequency_penalty | 整段删除（宿主控制） |
| **元叙述指令** | "作为 AI 助手"、"请根据设定"、"遵循系统提示" | 整段删除 |
| **大型嵌套关系** | 国家→城市→家族→个人→物品（> 2 层） | 扁平化为 1-2 层 |
| **纯百科条目** | 与角色/当前剧情无直接关联的完整历史/地理 | 删除，只保留相关片段 |

## 导入流程（agent 执行）

### 步骤 1：接收原始卡片

用户上传或粘贴外部卡片内容（ST PNG/JSON、AIRP 配置、纯文本等）。

### 步骤 2：自动解包与分类

agent 将内容拆分为四大块：
- **角色设定块**：外貌、性格、背景、说话风格、示例对话
- **文风/格式块**：句式规则、前缀后缀、修辞偏好、语言特征
- **世界观块**：背景知识、地理、组织、物品设定
- **机制块**：状态系统、骰子、战斗、经济、时间、代码、UI

### 步骤 3：机制块 → 直接删除

**机制块整块进入垃圾桶**，不保留。包括但不限于：
- 任何数值表格（HP、MP、STR、DEX 等）
- 任何规则说明（"当 HP < 20 时进入狂暴"）
- 任何计算逻辑（"伤害 = 攻击力 - 防御力"）
- 任何时间/经济系统（"每天获得 100 金币"）
- 任何脚本/代码（`if`、`for`、`function`）
- 任何 UI 描述（"左上角显示血量条"）

### 步骤 4：设定块 → 静态化清洗

对剩余三块内容进行清洗：

**角色设定块**：
- 删除 `{{user}}` 开场白（"你是她最好的朋友"）→ 改为人设描述
- 删除 `{{char}}` 自指 → 改为第三人称或角色名
- 删除情感状态数值（"好感度：50/100"）→ 改为性格描述（"对陌生人冷淡，对信任的人温和"）
- 删除示例对话中的机制指令（如 `<user>` 标签）→ 改为纯对话文本
- 保留：外貌、性格、背景、说话风格、动作习惯、纯对话示例

**文风块**：
- 删除门控逻辑（"当场景紧张时切换为短句"）→ 保留默认风格为短句
- 删除元叙述（"请用中文回答，保持角色一致"）→ 删除（preset 本身即风格指令）
- 删除多角色切换指令（"当 NPC-A 说话时..."）→ 删除（纯 Skill 单角色）
- 保留：句式规则、节奏、修辞、前缀后缀、语言特征、反例

**世界观块**：
- 删除门控（"仅当提到'龙族'时加载"）→ 保留为静态条目，删除门控语句
- 删除纯百科（"世界历史：公元前 3000 年..."）→ 只保留与角色直接相关的片段
- 删除多层嵌套（"帝国→行省→城市→家族→个人→装备"）→ 扁平为：帝国、城市、角色所在家族
- 删除数值设定（"魔法值消耗：10 点/次"）→ 改为描述性设定（"使用魔法会让她疲惫"）
- 保留：核心设定、与角色相关的背景条目、地理、冲突种子

### 步骤 5：重组为纯 Skill 三件套

将清洗后的内容填入三个模板，并按命名规范保存为 `SKILL.md` 文件：

```
原始卡片
  → 角色设定块 → 填入 character-SKILL.md 模板 → 保存为 character-<角色名>.md
  → 文风/格式块 → 填入 preset-SKILL.md 模板   → 保存为 preset-<文风名>.md
  → 世界观块   → 填入 worldbook-SKILL.md 模板 → 保存为 worldbook-<世界观名>.md
  → 机制块     → 删除
```

**命名规范**：
- 使用 `character-` / `preset-` / `worldbook-` 前缀，避免与其他 skill 冲突
- 后缀用角色名/文风名/世界观名（英文或拼音，无空格，如 `elara` / `whimsical` / `coven`）
- 示例：`character-elara.md`, `preset-whimsical.md`, `worldbook-coven.md`
- 同一角色的不同版本：加序号，如 `character-elara-v2.md`

**安装位置**：
- 在 Kimi Work 中，skill 文件放置于 agent 的 skills 目录下：
  ```
  <agent-skills-dir>/
  ├── character-elara/
  │   └── SKILL.md
  ├── preset-whimsical/
  │   └── SKILL.md
  └── worldbook-coven/
      └── SKILL.md
  ```
- 每个 skill 一个独立目录，目录名 = skill 名，内部固定为 `SKILL.md`
- 生成后，agent 可调用 `SkillManage` 或手动创建目录写入文件

**临时使用（不安装）**：
- 如果只是想跑一次、不想留文件：将三件套内容直接粘贴到 session 中作为 system prompt 或首轮上下文，不写入 skills 目录
- 缺点：无法复用、session 结束即丢失
- 优点：零残留、零管理成本

### 步骤 6：规模与触发检查

重组后检查：
- 角色卡：是否含 few-shot 对话范例？（无 → 补充 3 组）
- 预设：是否含文风锚和活文保障？（无 → 补充）
- 世界书：条目数是否 > 10？tokens 是否 > 1000？（是 → 精简）
- 整体：是否有不可实现内容残留？（是 → 回到步骤 3）
- **常驻层预算**：角色卡+预设合计是否 < 2500-3000 token？（超 → 把世界书移到懒加载，或上动态层）
- **`description` 触发力**：每份 skill 的 `description` 是否写了**触发场景 + 角色名/场景词/世界专名**，而非干巴巴一句"角色卡"？（决定相关性命中与手动可发现性）详见 [05 §4](05-loading-mechanism.md#4-description-是触发命门)。

### 步骤 7：输出与声明

输出三份 `SKILL.md` 文件（或目录），并在每份开头添加审计声明：

```markdown
> **审计声明**：本 skill 由外部卡片导入，已自动剔除不可实现内容（数值状态/骰子/战斗/文生图/HTML/代码/采样参数等）。原卡中的机制系统已删除，保留纯文字设定。如需完整机制（状态变化、战斗、经济），请使用动态层方案（MCP / AIRP）。
```

**安装后使用**（无"钉住"，见 [05-loading-mechanism.md](05-loading-mechanism.md)）：
1. 会话开场 /rp-launcher-<name>，由它统一拉起 character + preset + worldbook
2. 按 `docs/03-workflow.md` 步骤 6 输入首句剧情，开始 RP
3. 上下文被压缩后，重新 /rp-launcher-<name>

**卸载**：不需要时，直接删除对应 skill 目录即可：
- 删除 `<agent-skills-dir>/character-<name>/`
- 删除 `<agent-skills-dir>/preset-<name>/`
- 删除 `<agent-skills-dir>/worldbook-<name>/`
- 删除后该 skill 不再可被调用；若用了 CLAUDE.md 升级，从中删掉对应段落

**更新/替换**：
- 直接覆盖原目录中的 `SKILL.md` 文件 → 保留目录名，替换内容
- 或创建新版本目录（如 `character-elara-v2/`），然后删除旧版
- 建议先备份（重命名目录为 `character-elara-backup/`）再替换

## 常见 ST 卡片结构映射

| ST 卡片节 | 纯 Skill 归属 | 处理说明 |
|-----------|--------------|---------|
| Name / 角色名 | character-SKILL.md `name` | 直接保留 |
| Description / 角色描述 | character-SKILL.md 外貌+性格 | 删除其中的数值/机制 |
| First Message / 首句 | **删除** | 纯 Skill 首句由用户临场输入 |
| Personality / 性格 | character-SKILL.md 性格 | 静态化，删除数值 |
| Scenario / 场景 | character-SKILL.md 当前场景 + worldbook 地理 | 拆分 |
| Example Messages / 示例对话 | character-SKILL.md few-shot 范例 | 删除机制指令，保留纯对话 |
| Creator's Notes / 作者备注 | 视内容而定 | 机制 → 删除；设定 → 归入世界观 |
| System Prompt / 系统提示 | preset-SKILL.md 核心指令 | 删除元叙述和采样参数 |
| Post-history Instructions / 后文指令 | **删除** | 纯 Skill 无历史管理 |
| Character's Book / 角色世界书 | worldbook-SKILL.md | 静态化，删除门控 |
| Lorebook / 全局世界书 | worldbook-SKILL.md | 只保留相关条目 |
| Alternate Greetings / 备用开场 | **删除** | 纯 Skill 无开场管理 |
| Character's Tags / 标签 | **删除** | 用于分类，不进入 skill |
| Depth Prompt / 深度提示 | preset-SKILL.md 语言特征+反例 | 拆分归入文风 |
| Wildcard / 随机内容 | **删除** | 需要随机引擎，纯 Skill 无此能力 |
| Group / 群聊配置 | **删除** | 需要多 agent 调度 |
| World Info / 世界信息 | worldbook-SKILL.md | 静态化清洗 |

## 降级声明模板

当原始卡片被大幅剔除（> 50% 内容删除）时，在 skill 输出中附加：

```markdown
> ⚠️ **降级声明**：原始卡片包含大量机制系统（状态/战斗/骰子/经济/代码等），超出纯 Skill 静态范围。本 skill 已自动剔除所有不可实现内容，仅保留纯文字设定和文风。当前保留内容约占原卡 30%。如需完整机制体验，请使用动态层方案（MCP / AIRP / evented runtime）。
```

## 与 tavern2agent 的关系

`tavern2agent`（Xerxes-2/tavern2agent）是一个完整的卡片语义提取 + evented runtime 生成工具。它：
- 将 ST 卡片解包为语义 IR（`card-ir.json`）
- 生成状态机、reducers、event packs、CodeAct API
- 支持多 agent、秘密边界、两段式渲染

**纯 Skill 方案只借用它的"解包"和"审计"思路，不采用其 runtime**：
- 借鉴：从卡片提取结构、分类内容、识别机制与设定的思路
- 不采用：state schema、reducers、event packs、CodeAct、subagents、两段式渲染
- 本质差异：`tavern2agent` 是**动态层**（runtime），纯 Skill 是**静态层**（prompt-only 退化形态）

如果用户需要 `tavern2agent` 的完整能力（战斗、经济、多 agent、长战役），应直接迁移到该方案，而非在本项目硬塞机制。

## 审计检查清单（agent 自检）

重组完成后，agent 必须确认：
- [ ] 所有数值状态系统已删除（无 HP/MP/好感度/经验值）
- [ ] 所有骰子/战斗/经济/时间系统已删除
- [ ] 所有文生图提示词已删除
- [ ] 所有 HTML/CSS/UI 描述已删除
- [ ] 所有 ST 宏（`{{char}}`、`{{user}}`）已替换为静态文本或删除
- [ ] 所有 JSON Patch / 结构化数据指令已删除
- [ ] 所有 COT 标签已删除（除非角色在思考）
- [ ] 所有多 agent / NPC 协同指令已删除
- [ ] 所有秘密边界门控已删除（保留内容作为静态条目）
- [ ] 所有记忆/归档指令已删除
- [ ] 所有可执行代码已删除
- [ ] 所有采样参数已删除
- [ ] 所有元叙述指令已删除
- [ ] 世界书条目 ≤ 10 条，tokens ≤ 1000
- [ ] 角色卡含 few-shot 对话范例（≥ 3 组）
- [ ] 预设含文风锚和活文保障
- [ ] 角色卡+预设合计 < 2500-3000 token（常驻层预算）
- [ ] 每份 skill 的 `description` 含触发场景 + 专名（命中力 / 可发现性）
- [ ] 已生成元启动器 rp-launcher-<name>，且填入了三个 skill 名
- [ ] 三件套整体无机制残留

全部勾选 = 审计通过。任一未勾选 = 回到对应步骤重新清洗。
