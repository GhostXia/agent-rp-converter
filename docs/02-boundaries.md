# 硬边界：什么时候纯 Skill 会翻车

> **核心原则**：本方案是**纯文字、无状态、轻量级**的 RP 演绎。任何需要动态计算、持久化存储、跨会话记忆或多代理协同的机制，都必须**自动剔除**。

## 自动剔除清单（不可实现内容）

当从外部角色卡（SillyTavern / AIRP / 其他格式）导入时，agent 必须**自动剥离**以下内容。不提示用户，不保留，不降级——直接删除：

| 类型 | 示例 | 剔除原因 |
|------|------|----------|
| **数值状态系统** | HP/MP/好感度/经济/经验值动态变化 | 纯 Skill 无状态机，无法计算和持久化 |
| **骰子/战斗/经济系统** | 1d20、伤害计算、掉落表、技能冷却 | 需要确定性随机和规则引擎，纯 Skill 无此能力 |
| **时间推进系统** | 回合制、日期推进、季节变化机制 | 需要外部时钟或状态累加器 |
| **文生图提示词** | Stable Diffusion tags、{{style}}、图片生成指令 | 纯文字方案，不生成图片 |
| **HTML/CSS/前端面板** | 状态栏 HTML、颜色代码、进度条、UI 布局 | 纯文字，无界面 |
| **ST 宏/运行时补丁** | `{{char}}`、`{{user}}`、`<>` 替换语法、脚本注入 | 这是宿主运行时语法，不进入 Skill 正文 |
| **JSON Patch / 状态更新格式** | 结构化状态 diff、数据交换指令 | 无状态层，无数据可交换 |
| **COT 标签（思考链）** | `<think>`、`<reasoning>`、显式步骤拆解 | 破坏叙事沉浸，除非角色本身在思考 |
| **多 agent / NPC 协同** | 多角色并行调度、subagent 分工、阵营隔离 | 需要动态运行时和多会话架构 |
| **秘密边界/视角隔离** | 不同角色知道不同信息的门控机制 | 需要运行时状态管理和信息过滤层 |
| **记忆/封卷/跨会话归档** | "记住用户名字"、剧情日志自动归档 | 无持久化存储 |
| **可执行代码/脚本** | JavaScript、Python、条件判断逻辑 | Skill 是静态 markdown，不执行代码 |
| **采样参数** | temperature、top_p、frequency_penalty | 宿主控制，不进入 Skill |

**剔除原则**：
- 如果一段内容属于上表任何一类 → **删除整段**
- 如果一段内容混合了可保留设定（如角色外貌）和不可实现机制（如 HP 条）→ **删除机制部分，保留设定部分**
- 如果世界书条目包含"门控逻辑"（仅当 X 时加载）→ **删除门控，保留内容作为静态条目**
- 如果角色卡包含"首句用户台词"（{{user}} 开场）→ **删除，首句由用户临场输入**

## 绝对禁区（已剔除后仍不可引入）

以下需求**纯 Skill 无法满足**，即使已剔除清单内容，也必须确认不引入：

### 1. 活体状态（Living State）
- HP / MP / EXP 每轮变化
- 物品栏增减
- 技能冷却计时
- 好感度动态积累

> **Why**：Skill 是静态文件，不能记录状态。让模型自己估算数值会导致漂移和幻觉。

### 2. 跨会话记忆
- "上周剧情里我救过谁"
- "角色记得我偏好红茶"
- 长战役的伏笔回收

> **Why**：Skill 没有持久化存储，session 结束即清空。模型上下文窗口有限，> 20 轮后早期细节会被遗忘。

### 3. 大世界书（Big Lore）
- 200+ 条背景条目
- 关键词门控（只有提到"龙族"才加载龙族设定）
- 多层嵌套关系（国家→城市→家族→个人）

> **Why**：整本常载 → Token 爆炸。缺失门控 → 角色"全知"（知道不该知道的事），破坏沉浸。

### 4. 长战役（Long Campaign）
- > 20 轮对话
- 复杂分支剧情
- 需要追踪伏笔、NPC 关系网、时间线

> **Why**：上下文稀释。模型会遗忘早期设定，开始编造。

---

## 灰色地带（可妥协）

| 需求 | 纯 Skill 能撑多久 | 建议 |
|------|------------------|------|
| 简单数值（如好感度 0-100） | 5-10 轮 | 模型能记住，但会漂移。建议用户每轮人工确认 |
| 中型世界书（10-30 条） | 可以 | 全部载入，不加门控 |
| 中篇 RP（10-20 轮） | 可以 | 在 preset 里加"每 5 轮主动总结一次当前状态"的规则 |
| 轻微状态变化（情绪起伏） | 可以 | 交给模型自然演绎，不量化 |

---

## 翻车预警信号

运行中若出现以下情况，说明已触及边界，需要上动态层：
- 🚨 角色开始"全知"（知道未提及的设定）
- 🚨 数值前后矛盾（上轮 HP 20，这轮变成 50）
- 🚨 文风突然变成说明书（上下文被其他指令污染）
- 🚨 角色性格突变（早期设定被后期对话淹没）
- 🚨 设定冲突（世界书说 A，角色说 B）

---

## 决策树

```
你要跑什么 RP？
  ├── 单次 / 短对话（< 10 轮）
  │     ├── 无数值 → 纯 Skill ✅
  │     └── 有简单数值 → 纯 Skill 可尝试，注意漂移 ⚠️
  ├── 中篇（10-20 轮）
  │     ├── 小型世界 → 纯 Skill + 主动总结规则 ⚠️
  │     └── 中型世界 → 建议上动态层 ❌
  └── 长篇战役（> 20 轮）/ 有状态 / 大世界
        └── 必须上 MCP / AIRP ❌
```

## 总结

纯 Skill 方案是**轻量武器**，不是**万能工具**。它的定位是：

- **纯文字**：不生成图片、不渲染 UI、不执行代码
- **基础演绎**：无状态机、无战斗系统、无骰子、无经济
- **自动审计**：导入外部卡时，agent 必须自动剔除不可实现内容

- 适合：即兴 RP、短会话、无数值、小世界、文风实验
- 不适合：跑团、游戏化 RP、长篇小说、复杂状态管理

知道自己武器的射程，才不会在错误场景扣扳机。
