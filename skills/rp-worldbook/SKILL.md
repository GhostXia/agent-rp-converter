# 世界书模板

## 自动剔除（不可实现内容）

当从外部世界书（SillyTavern / AIRP / 其他格式）导入时，**必须自动剥离**以下内容。不保留，不降级，直接删除：

- 数值状态系统（HP/MP/好感度/经济/经验值动态变化机制、属性数值表）
- 骰子/战斗/经济/时间推进系统（任何需要计算的机制、回合规则、技能冷却表）
- 文生图提示词（Stable Diffusion tags、场景描述用于图片生成）
- HTML/CSS/前端面板（地图 UI、关系图、状态栏）
- ST 宏/运行时补丁（`{{char}}`、`{{user}}`、`<>` 替换语法、条件脚本）
- JSON Patch / 状态更新格式 / 结构化数据交换指令
- COT 标签（显式思考链）
- 多 agent / NPC 协同指令（多角色并行调度、subagent 分工）
- 秘密边界/视角隔离机制（"仅当角色 A 在场时显示"、隐藏信息门控）
- 记忆/封卷/跨会话归档指令
- 可执行代码/脚本（JavaScript、Python、条件逻辑）
- 采样参数（temperature、top_p 等）
- 大型嵌套关系（国家→城市→家族→个人，超过 2 层嵌套）
- 纯百科条目（与角色/当前剧情无直接关联的设定）

**处理原则**：
- 纯机制（如魔法消耗公式）→ **整段删除**
- 设定与机制混合（如世界观描述附战斗规则）→ **删除机制，保留设定**
- 门控逻辑（"仅当 X 时加载"、"仅对 Y 角色可见"）→ **删除门控，保留内容作为静态条目**
- 大型关系网 → **扁平化，只保留与角色直接相关的 1-2 层**
- 纯百科（如完整历史年表）→ **删除，只保留当前剧情需要的片段**

## 背景条目

按条目列出。每条 < 100 tokens，只写与角色/剧情直接相关的知识。

**条目格式**：
```
### <条目名>
触发词：<提到哪些词时加载此条目>
内容：<设定>
角色知道的程度：<全部/部分/仅传闻/不知道>
```

> ⚠️ 纯 Skill 无真正"门控"。本 skill 被**调用那一次**整本正文进上下文，之后留存到压缩——不是按关键词逐条自动加载/卸载。"触发词"字段仅提示模型**何时优先引用**此条目，不触发任何加载行为。逐轮关键词门控需动态层。详见 `docs/05-loading-mechanism.md`。
> 如果总条目 > 10 条或总 tokens > 1000，请拆分多个世界书 skill 或转动态层方案。

角色知道的程度：Elara 曾是成员，10 年前退出。知道部分内幕，但不知道现任成员名单。
